# スワイプボクシング

スマホ向け・一人称視点の3Dボクシングゲーム。Three.js製の軽量3Dシーンと、
Node.js製のオンライン対戦サーバーで構成されています。

## 実装済みの機能

1. ボクシングゲーム / 一人称視点 / 3D（Three.js、ビルド不要のブラウザネイティブES Modules）
2. スワイプ操作 — スワイプした画面の左右で利き手、方向でパンチの種類を判定
   - 上方向スワイプ = ストレート
   - 斜め上スワイプ = アッパー
   - 横方向スワイプ = フック
   - スワイプの速さ・距離がパンチの威力になります
3. 長押しでガード（ブロック）— 指を動かさず約200ms押し続けるとガード姿勢になり、
   被ダメージが約80%軽減されます
4. HPが0になった時点で確率判定：成功すれば低HPで復活、失敗すればKO。
   ダウンを重ねるほど次に復活できる確率が下がっていく（＝KO確率が上がる）仕様
   （`shared/matchLogic.js` 参照）
5. オンライン対戦（WebSocketによるサーバー中継・サーバー権威型のHP判定）

## ディレクトリ構成

```
client/          静的配信されるゲーム本体（HTML/CSS/JS、ビルド不要）
  index.html
  src/           シーン・入力・HUD・AI・ネット通信などのモジュール
  vendor/        three.module.js を同梱（CDN障害・オフライン環境でも動くように）
server/          Node.js製のオンライン対戦サーバー（Express + ws）
  src/index.js   静的配信 + WebSocketのマッチング/中継
  src/room.js    対戦ルームの状態管理（サーバー側が権威）
shared/          クライアント/サーバー共通のルール（ダメージ計算・復活/KO判定）
  matchLogic.js
```

## ローカルでの動かし方

```bash
cd server
npm install
npm start
```

`http://localhost:8787` をブラウザで開くとメニューが表示されます。
「CPUと練習」でオフライン対戦、「オンライン対戦」でオンライン対戦を選べます。

### 自分のスマホで試す（同じWi-Fi内）

1. PCとスマホを同じWi-Fiに接続
2. PCのLAN IP（例: `192.168.x.x`）を確認し、スマホのブラウザで
   `http://192.168.x.x:8787` を開く
3. 2台のスマホから同じアドレスにアクセスし、両方で「オンライン対戦」→
   同じ合言葉を入力すればその2台だけでマッチングします（何も入力しなければ
   早い者順の自動マッチングになります）

## ⑧ オンライン対戦（他のスマホとの対戦）の実現方法について

**採用した方式：WebSocketによるサーバー中継 + サーバー権威型の判定**

- 各プレイヤーのスマホは、Node.jsサーバーに対して1本のWebSocket接続を張るだけです。
  パンチやガードの操作はサーバーに送られ、サーバー側でHP・ダウン・KOを計算してから
  両方のスマホへ結果を配信します（`server/src/room.js`）。
- この方式（P2Pではなくサーバー経由）を選んだ理由は、キャリア回線やモバイルWi-Fi
  ではNAT越え（WebRTCのP2P直接接続）が不安定になりやすく、サーバー中継なら
  ほぼ確実にどんな2台のスマホ同士でも対戦できるためです。
- HP計算をサーバー側で行うことで、クライアント側の改造による不正なダメージ申告も
  防げます。

**「本当に離れた場所にいる他人のスマホ」と対戦するために必要なこと**

このセッションのサーバーは開発用コンテナ内で動いており、外部のスマホから直接は
アクセスできません。また、アカウント作成が必要なホスティングサービスへのデプロイは
本人にしかできない操作のため、こちらで自動では行えません。以下の手順で無料・数分で
公開できます。

### Renderへのデプロイ手順（無料・カード登録不要）

このリポジトリ直下に `render.yaml` を用意済みなので、Renderの「Blueprint」機能で
ワンクリックデプロイできます。

1. https://dashboard.render.com/ を開き、GitHubアカウントでログイン
2. 「New +」→「Blueprint」を選択
3. このリポジトリ（`hsgwyuki0429-design/kick`）を選択（初回はRenderにリポジトリへの
   アクセスを許可）
4. 内容を確認して「Apply」。数分でビルド・起動が完了し、
   `https://swipe-boxing-xxxx.onrender.com` のような公開URLが発行されます
5. そのURLを友達に送って、お互いのスマホで開けば対戦できます

（無料プランは一定時間アクセスがないとスリープし、次のアクセス時に起動まで
数十秒かかることがあります。頻繁に遊ぶ場合は有料プランへの変更で解消できます）

デプロイ後、`client/src/netClient.js`は自動的に接続元と同じホスト（`wss://` or
`ws://`）に繋ぐようになっているので、コードの変更は不要です。

Railway / Fly.io / 自前VPSなど別のサービスを使いたい場合も、`server`フォルダを
`npm install && npm start`で起動するだけの構成なので同様にデプロイできます。
希望があればそちらの設定ファイルも用意します。
